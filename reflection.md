## Reflection and Ethics

This system has some limitations and potential biases. It relies heavily on exact matches for genre and mood, which means it may not recommend songs that are similar in style but labeled differently. The dataset is also small, which limits diversity and may bias the recommendations toward certain genres.

This AI system could be misused if users rely on it too heavily or assume it always provides the best recommendations. To prevent this, the system provides explanations for each recommendation so users understand how decisions are made and can judge the results themselves.

During testing, one surprising result was that when no genre or mood matched, the system relied almost entirely on energy similarity. This showed that the system can still produce results, but they may be less accurate without strong feature matches.

While working with AI tools like Copilot, I found them helpful for generating code structure and improving readability. For example, Copilot helped generate parts of the scoring logic and test cases. However, one limitation was that some suggestions were too complex or did not match my project requirements, so I had to review and adjust the code to make sure it worked correctly.

Overall, this project helped me understand both the strengths and limitations of AI systems, and the importance of making them transparent, testable, and understandable.
